# Проект

Тут наши лабы по прекрасному предмету – хайлоду. Помни, орк, красный деплой едет быстрее, для сборки пользуйся Warforge.

## **Warforge**

**Warforge** — это твоя личная кузница войны и автоматизации.
Инструмент призывает (или уничтожает) контейнерные армии с помощью простых команд.
Больше никаких заклинаний `docker compose up/down` — только мощь, стиль и порядок на поле DevOps.

---

### **Идея**

Warforge — лёгкая CLI-обёртка над `docker compose`, созданная для проектов, где микросервисы растут быстрее, чем ты успеваешь помнить их имена.
Она помогает тебе **призывать воинов (сервисы)** или **уничтожать их** одним движением руки.

---

### **Команды**

#### **Призыв армии**

```bash
warforge march [-w warriors...] [-r]
```

* `-w`, `--warriors` — имена сервисов, которых нужно призвать.
  Если не указаны — призываются все.
* `-r`, `--rearm` — пересоздать контейнеры с нуля (`--force-recreate`).

**Примеры:**

```bash
warforge march
# Все контейнеры восстанут из кузницы

warforge march -w user-service limits-reader
# Призывает только user-service и limits-reader

warforge march -r
# Полный пересбор армии
```

---

#### **Уничтожение армии**

```bash
warforge slay [-w warriors...]
```

* `-w`, `--warriors` — имена сервисов для остановки.
  Если не указаны — все будут отправлены на покой.

**Примеры:**

```bash
warforge slay
# Тишина. Никого не осталось.

warforge slay -w user-service
# Уничтожен только user-service
```

### Список наших сервисов

Для удобства использования собрал тут наш зоопарк:
- `user-service` - сервис для управления пользователями
- `users-db` - база данных пользователей
- `users-db-migrations` - миграция базы данных пользователей
- `limits-db` - база данных rate limits
- `limits-writer` - писатель в базу rate limits
- `limits-reader` - читатель базы rate limits
